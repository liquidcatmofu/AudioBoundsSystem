# Audio Bounds System (ABS) & TTS Addon 総合設計仕様書

本ドキュメントは、マインクラフトにおける高度な音響空間制御Mod「**Audio Bounds System (ABS)**」を基盤とし、外部音声合成エンジンを管理する「**ABS TTS Addon**」、および各鉄道Mod（MTR, Create）と連携する拡張アドオンからなる、モジュール分離型音声基盤の統合設計仕様書である。

---

## 1. モジュール分離アーキテクチャ

システム全体を「再生・物理制御層（ABS基盤）」と「音声生成・トランスコード層（TTS Addon）」に分離し、鉄道アドオンがこれらを経由して機能する。これにより、TTSが不要な環境でもABS単体の高性能スピーカーシステムとして利用できる。

## 2. Audio Bounds System (ABS基盤) の仕様

ABS本体は「マインクラフトにおける高機能サウンド再生フレームワーク」として機能し、音声合成に関わるロジックは持たない。既存のOggファイル再生や、他のModからの再生要求を処理する。

### 2-1. 基盤ブロック・アイテム
* **スピーカーブロック (Speaker Block)**
    * 3D空間オーディオの位置基準および「音響バリア（減衰エリア）」の原点となるブロック。
    * エリア形状（Box等）と減衰カーブを設定し、隣接エリアへの音漏れを防ぐ。
* **オーディオコントローラー (Audio Controller)**
    * 基盤Mod単体で音声を制御・再生するための司令塔ブロック。
    * 隣接するレッドストーン（RS）信号入力の強度（1〜15）を検知し、TOMLで定義された台本やOgg音源を対象スピーカーから再生する。
    * OP権限で右クリックすると、ゲーム内管理GUIが開く。

### 2-2. 音声再生モードと音響バリアの設計
4つの再生モードを提供する。

#### ① 固定3D空間オーディオ（スピーカーモード & エリアバウンディング）
特定の境界を越えたら音を減衰させる「エリア制御型3D再生」。音源座標は常にスピーカーの実座標に固定し、クライアント側で位置に応じた音量（Volume）補正のみを行う。
* **エリア形状 (Audio Bounds Shape):** `Sphere`（球体）, `Box`（直方体）, `Cylinder`（円柱）, `Hemisphere`（半球）から選択。
* **フェードアウト (Falloff):** 境界から外側に向けて設定された幅（例: 3ブロック）でフェードアウトし、エリア外では無音になる。

#### ② 聴覚特性に基づく減衰カーブ（Attenuation Curves）
フェードアウト時の音量変化 $V(d)$ は、境界進入深度 $d \in [0, 1]$ に基づき計算される。
* **A. 線形（Linear）減衰:** $V(d) = 1.0 - d$
* **B. 対数知覚（Logarithmic）減衰 [推奨]:** ウェーバー・フェヒナーの法則に準拠。最小到達音量 $L_{\text{min}}$ (例: -60dB) として、$V(d) = 10^{\frac{L_{\text{min}} \cdot d}{20}}$
* **C. スムーズステップ（SmoothStep）減衰:** $V(d) = 1.0 - (3d^2 - 2d^3)$
* **D. 逆2乗物理（Inverse Square）減衰:** $V(d) = \frac{1.0}{(1.0 + k \cdot d)^2}$

#### ③ 動的座標追従型3Dオーディオ（移動音源）
* 列車（MTR/CRN）の車外スピーカー向け。Minecraft標準の `TickableSoundInstance` を実装し、毎ティック再生座標をエンティティに更新する。ドップラー効果やSPRの動的遮蔽が適用される。

#### ④ 2D/位置情報なし再生（車内案内・UI再生モード）
* 列車内の車内放送向け。減衰なし（`Attenuation.NONE`）、位置情報なし（`isRelative() = false`）で再生し、全方位から均一に聞こえるUI音として処理する。

### 2-3. ネットワーク配信とキャッシュ
* **内蔵HTTPServer:** ゲームポートとは別に、独立ポート（例: `25566`）で軽量HTTPServer（Netty等）を起動。ワンタイムトークン認証を用いてセキュアにOggをストリーミング配信する。
* **音源連結 (Queueing):** クライアント側でハッシュ配列（例: チャイム + アナウンス）を順次再生するPlaylist機能を実装。

### 2-4. 字幕同期と描画システム
* タイミングデータ（`SubtitleTrack`）をパケットで受信。クライアントの実再生経過ミリ秒を計算し、`SubtitleEvent` を抽出。
* `InGameGui` 描画イベントにフックし、Mod独自の透過度の高いモダンな字幕オーバーレイHUDとしてレンダリングする。

## 3. ABS TTS Addon (音声合成拡張) の仕様

ABS基盤を前提とし、テキスト情報を音声に動的変換して供給する。

### 3-1. 外部エンジンの抽象化と動的スキーマ
各TTSエンジンが独自の調声パラメータをGUIに提供できる「動的パラメータ・スキーマ」を実装する。

```java
public interface TTSProvider {
    String getId();
    // エンジンが要求する調声パラメータの定義（GUI動的生成用）
    List<VoiceParameter> getParameterSchema();
    // 音声合成および字幕タイミング（SubtitleTrack）の生成
    CompletableFuture<SynthesisResult> synthesize(String text, String speakerId, Map<String, Object> params);
    boolean isAvailable();
}
```

### 3-2. FFmpeg トランスコーディング
* Javaの `ProcessBuilder` を用い、VOICEVOX等から得たWAVを非同期で Ogg Vorbis に変換し、ABSのキャッシュシステムに登録する。

## 4. 鉄道アドオンModの独立設計（連携方針）

* **MTR連携アドオン (ABS - MTR Integration):** Mixinを使用し、MTR内部のTTS再生要求メソッドをインターセプト。標準の処理をキャンセルし、ABSの再生APIにリダイレクトする。
* **CRN連携アドオン (ABS - Create Integration):** Create Railway Navigator のデータ層から次の停車駅等の運行情報を取得し、TOMLテンプレートを介して音声合成・再生を行う。

## 5. TOML設定とGUI・コメント保持システム

ユーザー設定は、コメント（`#`）をサポートする **TOML形式** で統一する。

### 5-1. ディレクトリ構造とTOML定義例
```toml
# config/abs/controllers/main_lobby.toml
controller_id = "controller_main_lobby"

# ターゲットスピーカーと音響空間定義
[[target_speakers]]
speaker_id = "lobby_speaker_1"
relative_position = [0, 5, 0]
  [target_speakers.audio_bounds]
  shape = "box"
  size = [30.0, 6.0, 10.0]
  falloff_distance = 3.0
  attenuation_curve = "logarithmic"

# TTSプロバイダーごとに異なる動的調声パラメータ
[defaults.tuning]
speedScale = 1.05
pitchScale = 0.0

# RS信号強度 (1~15) に対応する再生アクション
[rs_triggers.1]
text = "ピンポンパンポン。お知らせいたします。"
queue = ["chime_lobby"]
```

### 5-2. コメント保持とGUI同期 (TOML Exchange)
* **検証:** サーバー側でデータサイズ上限（64KB）とTOML構文パースのみの安全な検証を行う。
* **コメント保持:** NightConfig 等のライブラリを活用し、GUIから値を上書き保存（シリアライズ）しても、既存のファイル内コメントが失われない構成管理を実現する。
* **GUIモックアップ:**

```text
+------------------------------------------------------------------------+
|  [ABS - スピーカー音響エリア設定]                                     |
+------------------------------------------------------------------------+
|  [一般設定]   [音響エリア形状設定]   [減衰カーブ設定]   [インポート/エクスポート] |
+------------------------------------------------------------------------+
|  エリア形状選択: [ 直方体 (box)                               ▼]      |
|                                                                        |
|  ■ 形状パラメータ:                                                     |
|  X軸幅 (Width) : [ 30.0   ]  Y軸高 (Height) : [ 6.0    ]              |
|  Z軸奥 (Depth) : [ 10.0   ]                                            |
|                                                                        |
|  ■ 境界減衰およびカーブモデル設定:                                      |
|  減衰カーブモデル: [ 対数知覚 (logarithmic)                   ▼]      |

|                                                                        |
|  [ 適用して保存 ]                                       [ キャンセル ] |
+------------------------------------------------------------------------+
```

## 6. Sound Physics Remastered (SPR) 互換性
* **固定3D再生:** クライアント側でエリア減衰計算を行った後の `volume` を適用し、座標はスピーカーに固定してOpenALへ渡す。SPRは正常に反響や遮蔽を計算する。
* **動的追従3D再生:** `TickableSoundInstance` により毎ティック座標が更新されるため、移動音源としてSPRの物理計算が適用される。
* **2D音源再生:** 位置情報を持たないため、SPRによる計算対象から自動的に除外される。

## 7. 開発フェーズ（ロードマップ）

* **Phase 1: TTS Addon 単体構築**
    * VOICEVOX通信、WAV取得、FFmpegトランスコード（WAV $\rightarrow$ Ogg）の実装。`SubtitleTrack` の生成検証。
* **Phase 2: ABS基盤 - 内蔵HTTPServerの実装**
    * Ogg配信サーバーの構築と、ワンタイムトークン認証の検証。
* **Phase 3: ABS基盤 - 3D/2D再生とエリアバウンディング**
    * スピーカーブロックの登録。形状判定（Box等）と4種の減衰カーブ（Logarithmic等）の演算ロジック実装。
    * SPR環境下での音量事前適用・座標固定の動作検証。
* **Phase 4: ABS基盤 - HUD字幕とTOML・GUIシステム**
    * 独自透過HUD字幕レンダラの構築。
    * コントローラーブロック登録。NightConfigを用いたコメント保持TOML同期とGUI実装。
* **Phase 5: アドオン連携とリリース**
    * MTRアドオン（MixinによるTTS置換）、CRNアドオンの開発。1.20.1 Forge / Fabric ベータリリース。
